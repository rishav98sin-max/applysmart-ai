## Additional Fixes (April 22, 2026 - Evening)

### 7. ChromaDB v0.6.0 Compatibility Fix
**File:** `agents/cv_embeddings.py` (line 165-166)

**Change:** Updated `list_collections()` call to handle Chroma v0.6.0 breaking change:
- Old: `existing = {c.name for c in client.list_collections()}` (expected collection objects)
- New: `existing = set(client.list_collections())` (handles string names only)

**Reason:** Chroma v0.6.0 changed `list_collections()` to return only collection names (strings) instead of full collection objects, causing `NotImplementedError`. This broke RAG vector indexing, causing fallback to full CV text instead of chunked retrieval.

---

### 8. PDF Bullet Character Extraction Fix
**File:** `agents/cv_parser.py` (function `parse_cv`)

**Change:** Changed PDF text extraction from default mode to dict mode:
- Old: `page.get_text()` (default mode)
- New: `page.get_text("dict")` with span-level extraction

**Reason:** Default PyMuPDF extraction was converting special bullet characters (→, ▸, •) to ? symbols due to encoding issues. The dict mode preserves character encoding better by extracting text at the span level, preventing garbled bullet characters in the parsed CV text.

**Impact:** This fixes the issue where tailored CVs had ? symbols instead of proper bullets because the LLM was working with broken source text from the PDF parser.
