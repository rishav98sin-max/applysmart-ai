# Changes Made - April 22, 2026

## 1. Semantic Matching Fix (job_matcher.py)
**File:** `agents/job_matcher.py` (line 388)

**Change:** Updated semantic matching rules to include explicit mappings for:
- "agent-based AI systems orchestration"
- "automation workflows"
- "APIs"
- "SaaS tools"
- "third-party machine learning products"

**Reason:** These skills were being incorrectly marked as "missing" even though the user's CV contains related terms like "AI", "LangGraph", "APIs", etc.

## 2. UI Stage Indicators
**Files:** `agents/job_agent.py`, `app.py`

**Changes:**
- Added `progress_callback` parameter to `run_agent()` function
- Added `progress_callback` field to `AgentState` TypedDict
- Added `_report_progress()` helper function to safely call progress callback
- Added `_check_stop_requested()` helper function to check for stop requests
- Added progress calls to all worker nodes:
  - `validate_inputs_node` → "Validating inputs"
  - `parse_cv_node` → "Parsing CV"
  - `extract_cv_style_node` → "Extracting CV style"
  - `planner_node` → "Planning search strategy"
  - `scrape_jobs_node` → "Scraping job boards (round N)"
  - `match_jobs_node` → "Matching jobs to CV"
  - `tailor_and_generate_node` → "Tailoring CVs and generating cover letters (N jobs)"
  - `send_email_node` → "Sending email with documents"
- Moved stage indicator outside `st.status` block in `app.py` for better visibility
- Stage displays as "**Current Stage:** [stage name]"

**Reason:** Users couldn't see what stage the agent was at during execution, leading to poor feedback.

## 3. Stop Button
**Files:** `agents/job_agent.py`, `app.py`

**Changes:**
- Added "⏹ Stop" button in `app.py` above the status box
- Button sets `st.session_state["_stop_requested"] = True`
- Added stop check in `supervisor_node()` in `job_agent.py`
- If stop requested, agent exits with "halted_user_stop" status
- Added "halted_user_stop" to `HARD_TERMINAL_STATUSES`

**Reason:** Users needed a way to cancel the agent if they made a mistake with job inputs, to save tokens.

---

## 4. LinkedIn Description Retry Logic
**File:** `agents/job_scraper.py` (function `_fetch_linkedin_description`)

**Change:** Added retry logic that:
- Attempts to fetch description twice
- Retries with 2-second delay if first attempt returns empty or <30 chars
- Logs retry attempts

**Reason:** Intermittent network issues or LinkedIn rate limiting were causing empty descriptions, leading to valid jobs being skipped.

---

## 5. Gemini API Key Debug Logging
**File:** `agents/llm_client.py` (function `_call_gemini`)

**Change:** Added debug logging that:
- Logs the length of the API key
- Logs the first 10 characters of the key
- Warns if key doesn't start with "AIza"

**Reason:** Gemini API key was showing "API_KEY_INVALID" error despite being set in secrets. Debug logging helps identify if the key is being loaded correctly.
